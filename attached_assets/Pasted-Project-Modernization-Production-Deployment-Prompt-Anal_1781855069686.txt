Project Modernization & Production Deployment Prompt

Analyze the entire codebase and transform it into a production-ready Car Wash & Auto Detailing Management System suitable for deployment on a Linux VPS using PostgreSQL.

Critical Rule

Do NOT remove existing business functionality.

Preserve and improve all current features including:

* Authentication and login
* Dashboard
* Inventory management
* Transactions
* Customer accounts
* Service management
* Invoicing
* Database operations
* Reporting
* Existing workflows

If a feature is broken, fix it rather than removing it.

⸻

Primary Goal

Convert this project into a modern, scalable, maintainable, and deployment-ready business management system for a professional car wash and auto detailing company.

The final application should be suitable for:

* VPS deployment
* PostgreSQL production database
* Multiple users
* Real business operations
* Future expansion

Architecture Requirements

Backend

Refactor backend using Flask best practices:

* Blueprint-based architecture
* Service layer separation
* Modular routes
* Reusable utilities
* Environment-based configuration
* Proper error handling
* Structured logging

Suggested structure:

backend/
├── app/
│   ├── routes/
│   ├── services/
│   ├── models/
│   ├── forms/
│   ├── utils/
│   ├── middleware/
│   └── templates/
├── static/
├── migrations/
├── config.py
└── app.py

⸻

Database

Migrate fully to PostgreSQL.

Requirements:

* SQLAlchemy models
* Alembic/Flask-Migrate migrations
* Proper relationships
* Foreign key constraints
* Indexes on frequently queried fields
* Production-safe schema

Remove SQLite dependency.

Use environment variables:

DATABASE_URL=postgresql://user:password@host:5432/database

⸻

Configuration

Create:

.env.example

Required variables:

SECRET_KEY=
DATABASE_URL=
FLASK_ENV=
SESSION_SECRET=
APP_URL=

No hardcoded credentials.

Refactoring Process

Step 1:
Analyze current project structure.

Step 2:
Identify broken code, duplicates, dead files, and architectural issues.

Step 3:
Create a modernization plan.

Step 4:
Implement changes incrementally.

Step 5:
Test every module.

Step 6:
Ensure application runs without errors.

Step 7:
Prepare production deployment package.

Do not rewrite blindly. Analyze first, preserve functionality, then modernize and prepare for production deployment.
